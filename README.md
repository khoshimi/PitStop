# Express + Sequelize (SQLite)

## Установка
```bash
npm install
```

## Запуск (dev)
```bash
npm run dev
```

## Запуск (prod)
```bash
npm start
```

Открывай сайт **только через сервер**: `http://localhost:3000/`

## Админ-панель
1. В `.env` задай `ADMIN_PHONE` (11 цифр, например `79990000000`), `ADMIN_NAME`, `ADMIN_PASSWORD` (≥ 6 символов).
2. При **первом** запуске, если пользователя с таким телефоном ещё нет, он создаётся с ролью `admin`.
3. На странице входа (`/reg.html`) войди **теми же** именем, телефоном и паролем — произойдёт переход на **`/admin.html`** (не на профиль).
4. В админке: список пользователей, редактирование/удаление отзывов, замена трёх картинок акций на главной.

Запросы к API админки идут с заголовком `Authorization: Bearer <JWT>` (токен сохраняется в `localStorage` как `pitstop_token` при входе).

## Регистрация и вход (пароль — bcrypt в БД)
- `POST /api/auth/register` — в ответе `user`, `token`
- `POST /api/auth/login` — в ответе `user`, `token`

## Отзывы (страница «О нас»)
- `GET /api/reviews` — публично
- `POST /api/reviews` — добавить отзыв (форма на сайте)

## Акции на главной
- `GET /api/promos` — три URL картинок
- `PUT /api/admin/promos` — загрузка файлов (поля `image0`, `image1`, `image2`), только админ

## Прочее
- `GET /health`
- `POST /todos`, `GET /todos`

## Если после обновления кода ошибки SQLite / нет поля `role`
Останови сервер, удали `dev.sqlite` в корне проекта и запусти снова.

## Переменные `.env`
- `JWT_SECRET` — секрет подписи JWT (в продакшене — длинная случайная строка)
- `ADMIN_PHONE`, `ADMIN_NAME`, `ADMIN_PASSWORD` — учётка администратора
